## Summary

Returns TRUE when a number is odd.

## Remarks
 - Numeric input is truncated toward zero before parity is checked.
 - Booleans are coerced (`TRUE` -&gt; 1, `FALSE` -&gt; 0).
 - Non-numeric text returns `#VALUE!`.
 - Errors propagate unchanged.
